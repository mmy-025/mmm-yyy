# 一、数据清洗

## 1\. 知识讲解

数据清洗是数据分析的前置核心步骤，指对原始数据中的缺失值、重复值、异常值、无效格式、冗余数据进行检测与修正，提升数据质量，保证后续分析结果准确。原始数据普遍存在脏数据，数据清洗通常占据数据分析60%\-70%的工作量。

**核心处理对象**：缺失值、重复值、格式错误值、无效数据、冗余字段。

## 2\. 核心语法（Python pandas）

```python
# 1. 查看数据基本信息
df.info()  # 查看缺失值、数据类型
df.describe()  # 查看数值型数据统计特征
df.duplicated()  # 检测重复值

# 2. 重复值处理
df.drop_duplicates(inplace=True)  # 删除重复行

# 3. 缺失值处理
df.isnull().sum()  # 统计各列缺失值数量
df.dropna()  # 删除含缺失值的行
df.fillna(value)  # 填充缺失值（均值/中位数/众数/固定值）

# 4. 格式修正
df['列名'] = pd.to_datetime(df['列名'])  # 时间格式转换
df['列名'] = df['列名'].astype(类型)  # 数据类型转换
```

## 3\. 操作步骤

1. **数据探查**：通过info、describe、isnull等方法，排查缺失、重复、格式问题；

2. **重复值处理**：删除完全重复或关键字段重复的无效数据；

3. **缺失值处理**：少量缺失值删除，大量缺失值根据字段特性填充（数值用均值/中位数，分类用众数）；

4. **数据修正**：统一时间、数值、字符串格式，删除无效字符、冗余字段；

5. **结果校验**：再次探查数据，确认无脏数据。

## 4\. 实战举例

电商订单数据清洗：订单表存在用户手机号缺失、订单时间格式为字符串、重复订单数据。

```python
import pandas as pd
df = pd.read_csv("order_data.csv")
# 1. 删除重复订单
df.drop_duplicates(subset=["order_id"], inplace=True)
# 2. 手机号缺失值填充为未知
df["phone"] = df["phone"].fillna("未知")
# 3. 时间格式转换
df["order_time"] = pd.to_datetime(df["order_time"])
# 4. 校验清洗结果
print(df.isnull().sum())
```

# 二、分组聚合分析

## 1\. 知识讲解

分组聚合是将数据集按照一个或多个维度**分组**，再对每组数据进行统计计算（求和、均值、计数、最大值、最小值等），实现维度拆解分析，是结构化数据分析的核心手段。常用于业务维度统计，如按地区、时间、品类统计销售额。

## 2\. 核心语法

```python
# 基础语法
df.groupby(分组字段)[聚合字段].聚合函数()
# 多聚合、多维度分组
df.groupby([字段1,字段2]).agg(新字段名=(原字段,聚合函数), 新字段名2=(原字段2,聚合函数2))
```

**常用聚合函数**：count\(\)计数、sum\(\)求和、mean\(\)均值、max\(\)最大值、min\(\)最小值、std\(\)标准差

## 3\. 操作步骤

1. 确定分析维度（分组字段）和分析指标（聚合字段）；

2. 使用groupby完成数据分组；

3. 调用聚合函数完成指标计算，多指标使用agg批量聚合；

4. 重置索引，优化数据展示格式；

5. 筛选、排序结果，输出分析结论。

## 4\. 实战举例

统计各省份、各品类的订单量和销售额均值

```python
# 多维度分组+多指标聚合
res = df.groupby(["省份","品类"]).agg(
    订单数量=("order_id","count"),
    平均销售额=("amount","mean")
).reset_index()
# 按订单数量降序排序
res = res.sort_values("订单数量",ascending=False)
```

# 三、购物篮分析

## 1\. 知识讲解

购物篮分析（关联规则挖掘），用于挖掘用户同时购买商品的关联关系，核心是找出**商品A购买后大概率购买商品B**的规律，应用于商品捆绑销售、货架摆放、推荐系统。

**三大核心指标**：

- 支持度\(Support\)：P\(A∩B\)，同时购买A、B的订单占总订单比例，反映关联的普遍性；

- 置信度\(Confidence\)：P\(B\|A\)=P\(A∩B\)/P\(A\)，购买A后再购买B的概率，反映关联的可靠性；

- 提升度\(Lift\)：P\(B\|A\)/P\(B\)，提升度\>1表示正相关，=1无关联，\<1负相关。

## 2\. 核心公式

支持度：$Support(A\to B) = \frac{同时包含A、B的交易数}{总交易数}$

置信度：$Confidence(A\to B) = \frac{同时包含A、B的交易数}{包含A的交易数}$

提升度：$Lift(A\to B) = \frac{Confidence(A\to B)}{Support(B)}$

## 3\. 操作步骤

1. 数据预处理：将订单数据转换为「订单\-商品」一维事务格式；

2. 数据独热编码：转换为布尔型矩阵（每个订单对应是否购买某商品）；

3. 设置阈值（最小支持度、最小置信度），挖掘关联规则；

4. 筛选高提升度、高置信度的有效规则；

5. 业务解读与落地应用。

## 4\. 实战举例

基于超市交易数据，挖掘商品关联购买规则

```python
from mlxtend.frequent_patterns import apriori, association_rules
# 1. 构造事务数据
transaction = df.groupby("订单号")["商品名称"].apply(list).tolist()
# 2. 独热编码
from mlxtend.preprocessing import TransactionEncoder
te = TransactionEncoder()
te_array = te.fit(transaction).transform(transaction)
df_trans = pd.DataFrame(te_array,columns=te.columns_)
# 3. 挖掘频繁项集（最小支持度0.05）
frequent_items = apriori(df_trans,min_support=0.05,use_colnames=True)
# 4. 生成关联规则（最小置信度0.2）
rules = association_rules(frequent_items,min_threshold=0.2)
# 5. 筛选有效正相关规则
valid_rules = rules[rules["lift"]>1]
```

# 四、客户聚类分析

## 1\. 知识讲解

客户聚类是**无监督机器学习算法**，核心是根据客户多维特征（消费金额、消费频次、购买周期等），将相似客户自动划分为不同群体，实现客户分层，用于精准营销、客户留存、用户运营。最常用算法为K\-Means聚类。

## 2\. 核心原理与公式

K\-Means核心：通过迭代最小化**簇内平方和SSE**，将数据划分为K个簇。

簇内平方和公式：$SSE=\sum_{i=1}^k\sum_{x\in C_i}(x-\mu_i)^2$

其中：k为聚类数，$C_i$为第i个簇，$\mu_i$为簇中心点均值。

## 3\. 操作步骤

1. 特征选取：筛选客户核心特征（RFM模型：最近消费时间、消费频次、消费金额）；

2. 数据预处理：标准化、归一化（消除量纲影响）；

3. 确定最优K值（手肘法、轮廓系数法）；

4. 训练K\-Means模型，完成聚类；

5. 各簇特征分析，给客户群体打标签（高价值客户、沉睡客户等）。

## 4\. 实战举例

基于RFM特征对电商客户分层聚类

```python
from sklearn.cluster import KMeans
from sklearn.preprocessing import StandardScaler
# 1. 选取RFM特征并标准化
features = df[["最近消费天数","消费频次","总消费金额"]]
scaler = StandardScaler()
features_scaler = scaler.fit_transform(features)
# 2. 训练模型，分为4类客户
kmeans = KMeans(n_clusters=4,random_state=42)
df["客户标签"] = kmeans.fit_predict(features_scaler)
# 3. 查看各聚类群体特征
print(df.groupby("客户标签")[["最近消费天数","消费频次","总消费金额"]].mean())
```

# 六、A/B测试分析

## 1\. 知识讲解

A/B测试是一种**对照实验统计方法**，将用户随机分为对照组（A组，原方案）和实验组（B组，新方案），通过统计检验判断新方案指标是否显著优于原方案，用于产品迭代、运营策略优化、页面改版等场景。

核心逻辑：通过假设检验，排除数据波动，确认差异为方案本身带来的有效差异。

## 2\. 核心公式与指标

适用检验：均值检验（t检验）、比率检验（卡方检验）

核心判定标准：**P值\<0\.05**，差异显著；P值≥0\.05，差异不显著。

提升率公式：$提升率 = \frac{实验组指标-对照组指标}{对照组指标}\times100\%$

## 3\. 操作步骤

1. 确定测试目标与核心指标（点击率、转化率、客单价等）；

2. 设计实验组、对照组，保证样本随机、无干扰；

3. 采集两组实验数据，清洗预处理；

4. 进行统计显著性检验（t检验/卡方检验）；

5. 计算指标提升率，输出实验结论（新方案是否上线）。

## 4\. 实战举例

对比新旧页面转化率是否存在显著差异

```python
from scipy.stats import chi2_contingency
# 对照组、实验组转化数据：[转化数, 未转化数]
group_a = [80, 920]
group_b = [105, 895]
data = [group_a,group_b]
# 卡方检验
chi2,p,df,expected = chi2_contingency(data)
print(f"P值：{p}")
if p < 0.05:
    print("差异显著，新页面转化率显著更高")
else:
    print("差异不显著，无法证明新页面更优")
```

# 七、时间序列分析

## 1\. 知识讲解

时间序列是基于**时间顺序排列的动态数据**的分析方法，核心挖掘数据的趋势、周期性、季节性、随机性，实现数据预测。广泛应用于销量预测、流量预测、金融数据预测。

时间序列四大组成部分：长期趋势\(T\)、季节波动\(S\)、周期波动\(C\)、随机扰动\(I\)。

## 2\. 核心模型与语法

常用模型：移动平均MA、指数平滑、ARIMA、SARIMA（适配季节性数据）

```python
# 时间序列基础处理
df["时间"] = pd.to_datetime(df["时间"])
df = df.set_index("时间")  # 设置时间索引
df.resample("M").sum()  # 按月重采样聚合

# ARIMA模型语法
from statsmodels.tsa.arima.model import ARIMA
model = ARIMA(数据,order=(p,d,q))  # p自回归阶数、d差分阶数、q移动平均阶数
result = model.fit()
result.predict(start,end)  # 预测
```

## 3\. 操作步骤

1. 数据预处理：时间格式转换、设置时间索引、缺失值填充；

2. 时序可视化：观察趋势、季节性、波动特征；

3. 平稳性检验（ADF检验），非平稳数据做差分处理；

4. 选择适配模型，训练模型；

5. 模型预测、误差评估、结果输出。

## 4\. 实战举例

基于月度销量数据预测未来3个月销量

```python
# 时间索引重采样
sale_month = df.resample("M")["销量"].sum()
# 训练ARIMA模型
model = ARIMA(sale_month,order=(1,1,1))
fit_model = model.fit()
# 预测未来3期数据
pred = fit_model.get_forecast(steps=3)
print("未来3个月销量预测：",pred.predicted_mean)
```

# 八、特征工程

## 1\. 知识讲解

特征工程是对原始数据进行加工、转换、衍生，生成高质量特征的过程，是机器学习建模的核心基础，特征质量直接决定模型效果。核心目的：让机器读懂数据、提升模型精度、降低拟合误差。

**核心分类**：特征提取、特征转换、特征衍生、特征筛选。

## 2\. 核心语法与方法

```python
# 1. 分类特征编码
pd.get_dummies(df["分类字段"])  # 独热编码
from sklearn.preprocessing import LabelEncoder
le = LabelEncoder() # 标签编码（有序分类）

# 2. 数值特征转换
StandardScaler()  # 标准化：均值0，方差1
MinMaxScaler()  # 归一化：压缩至0-1区间

# 3. 特征衍生
df["新特征"] = df["字段1"] / df["字段2"]  # 业务衍生特征
df["月份"] = df["时间"].dt.month  # 时间特征衍生

# 4. 特征筛选
df.corr()  # 相关性筛选，剔除高冗余特征
```

## 3\. 操作步骤

1. 特征梳理：区分数值型、分类型、时间型原始特征；

2. 特征清洗：过滤无效、缺失严重、方差为0的无用特征；

3. 特征转换：编码分类特征、标准化数值特征；

4. 特征衍生：结合业务生成新特征（如客单价=总金额/订单数）；

5. 特征筛选：剔除高相关冗余特征，保留核心有效特征。

## 4\. 实战举例

用户数据特征工程处理，适配用户流失预测模型

```python
# 1. 时间特征衍生
df["消费月份"] = df["消费时间"].dt.month
# 2. 分类特征独热编码
df = pd.get_dummies(df,columns=["用户等级","设备类型"])
# 3. 数值特征标准化
scaler = StandardScaler()
df["消费金额"] = scaler.fit_transform(df[["消费金额"]])
# 4. 衍生核心特征
df["月均消费"] = df["总消费金额"] / df["消费月数"]
```

# 九、异常值检测

## 1\. 知识讲解

异常值（离群点）是指偏离正常数据分布的极端数值，由数据错误、特殊业务场景、系统误差导致。异常值会干扰统计分析、模型训练，需检测并处理（保留/修正/删除）。

**常用检测方法**：3σ原则、箱线图四分位数法、Z分数法。

## 2\. 核心公式

**1\. 箱线图法（四分位数法，最常用）**

四分位距：$IQR = Q3 - Q1$

异常值边界：$下限=Q1-1.5\times IQR，上限=Q3+1.5\times IQR$

超出上下限的数据即为异常值。

**2\. 3σ原则**

数据服从正态分布时，$\mu\pm3\sigma$以外的数据为异常值

## 3\. 操作步骤

1. 选取需要检测的数值型字段；

2. 通过箱线图/3σ原则计算异常值边界；

3. 筛选出异常数据，定位异常样本；

4. 业务判定：业务合理异常（大额订单）保留，错误数据删除/修正；

5. 完成异常值处理。

## 4\. 实战举例

通过四分位数法检测销售额异常值

```python
# 计算四分位数
Q1 = df["销售额"].quantile(0.25)
Q3 = df["销售额"].quantile(0.75)
IQR = Q3 - Q1
# 计算异常值边界
lower = Q1 - 1.5*IQR
upper = Q3 + 1.5*IQR
# 筛选异常值
outlier = df[(df["销售额"]<lower)|(df["销售额"]>upper)]
print("异常数据：",outlier)
# 删除异常值
df_clean = df[(df["销售额"]>lower)&(df["销售额"]<upper)]
```

# 十、多数据集合并

## 1\. 知识讲解

多数据集合并是将多个结构相同/关联的数据集整合为完整数据集的操作，分为**纵向合并（拼接行）**和**横向合并（拼接列/关联匹配）**，是多源数据整合的基础操作。

纵向合并：适用于字段相同、数据分批的数据集；横向合并：适用于通过关键字段关联的不同维度数据集。

## 2\. 核心语法

```python
# 1. 纵向合并（行拼接）
pd.concat([df1,df2,df3],ignore_index=True)

# 2. 横向合并（关联匹配，类似SQL连表）
pd.merge(左表,右表,on=关联字段,how=连接方式)
# 连接方式：inner内连接、left左连接、right右连接、outer全连接
```

## 3\. 操作步骤

1. 探查多数据集结构，确认字段、数据类型、关联主键；

2. 统一字段名称、数据类型，消除格式差异；

3. 根据业务需求选择纵向拼接或横向关联；

4. 选择匹配方式，完成数据集合并；

5. 去重、校验数据完整性，输出完整数据集。

## 4\. 实战举例

合并月度订单表，并关联用户信息表

```python
# 1. 纵向合并1-3月订单数据
df1 = pd.read_csv("1月订单.csv")
```
